# Arcadium

Arcadium is a cross-platform arcade-cabinet frontend built with Godot 4 .NET and C#.

It is intended to run fullscreen on Windows and Linux, scan local ROM and media folders, and launch configured emulators from a controller-first interface.

## Repository layout

- `src/Arcadium.Shared`: shared domain models and future configuration/database contracts.
- `src/Arcadium.Scanner`: C# command-line ROM scanner.
- `src/Arcadium.Godot`: Godot cabinet interface.
- `config`: tracked JSON examples and built-in emulator profiles.
- `database/migrations`: SQLite schema migrations.
- `docs`: roadmap and architecture documentation.

## Development setup

1. Install the .NET 8 SDK.
2. Install the Godot 4 .NET editor for UI development.
3. Copy `config/cabinet.example.json` to `config/cabinet.json`.
4. Copy `config/systems.example.json` to `config/systems.json`.
5. Run:

   ```bash
   dotnet restore
   dotnet build src/Arcadium.Scanner/Arcadium.Scanner.csproj
   dotnet run --project src/Arcadium.Scanner -- --help
   ```

The Godot `.csproj` is generated by the Godot .NET editor when `src/Arcadium.Godot/project.godot` is opened.

## Configuration and data

Human-editable configuration is JSON. SQLite is reserved for scanned games, media links, play history, favorites, and scan logs.

Never commit ROMs, artwork collections, local database files, or real cabinet paths.

See [the roadmap](docs/roadmap.md) for the complete implementation plan.
